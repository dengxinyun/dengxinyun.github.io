git提交代码到coding的一些分享

2018-10-29
git

coding是国内的一个代码托管平台，是一个高效的研发流程与管理平台，它让多个同事同时维护一个项目更加方便了。
但是我在coding的道路上踩过很多坑。。。

【辅助网址：】
http://www.liaoxuefeng.com/wiki/0013739516305929606dd18361248578c67b8067c8c017b000/00137621280731812dec22ecc9b44f4b2ca1c680f181a5b000

=========================================================================

【克隆远程版本库（注意版本库的地址）】

git clone https://git.coding.net/liu2017/DDdingshui.git

=========================================================================

【拉取代码】

git pull

=========================================================================

【提交代码】

1.把文件加入到暂存区（修改、新增的文件）
git add . //全部
git add file.php //file.php文件

2.提交到本地版本库（注意版本说明必填）
git commit -m ‘版本说明’

3.推送至远程版本库
git push origin master

==========================================================================

【解决冲突】

1.先拉取远程版本库代码
1）git pull //拉取远程版本库代码，自动合并到本地代码

2) git fetch origin master:tmp
git diff tmp
git merge tmp //拉取远程版本库代码，不会自动合并（理论上此方法更安全）

2.git status //查看版本状态，查看冲突原因

3.打开冲突的文件，找到冲突的位置，如下：（以=====隔开，上面的是本地代码，下面的是服务器上的代码）
‘<<<<<<<’ HEAD
我是本地代码（2）
“=======”
我是服务器上的版本（1）
“>>>>>>>” 19ff362904a580b17634a007beb59b4ae9f7cc93

4.手动处理冲突，把<<<<<<<< >>>>>>>>>符号删掉

5.重新执行【提交代码】中的步骤

注意：解决冲突中的步骤中１中的１２步只需要选择其中一种操作

===========================================================================

【合并冲突】

git merge //合并冲突并没有解决冲突，而是把本地和服务器上的版本合并，然后可以推送到远程版本库

==========================================================================

【注意】

2.有时提交的时候没发现什么错误，却提交不上去，此时，查看该文件夹下是否有git的忽略文件，如果有，删除即可。

==========================================================================

rm -rf .git //删除本地分支 （本地分之乱码的情况）
git init //重新初始化git

==========================================================================

【本地创建分支提交coding】

git init //初始化一个分支

git remote add origin(本地分支的名字) 地址（coding上的地址）

git add -A //提交全部

git commit -m ‘注释’

git push origin(本地分支的名字) master:分支名 //把本地分支提交到coding上

删除分支 git push origin(本地分支的名字) -d front （不可以删除分支，会导致历史记录被清空）

切换分支：git checkout qianduan

查看有什么修改的代码：git status

==========================================================================

【clone分支】

克隆github上的仓库到本地，默认会把仓库的所有内容clone到本地。

cd delun（进入delun文件夹） git branch -r （查看分支所有分支名）

这时用 git checkout front(分支名) 操作就把远程分支取到本地。

这时再用不带-r的git branch命令就能看到刚才操作的分支名了。

==========================================================================

ls -a (查看当前有没有git文件)

git remote //查看本地分支
